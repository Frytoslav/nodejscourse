# Checklista postepow

Zaznaczaj elementy dopiero wtedy, gdy umiesz je wyjasnic wlasnymi slowami i odtworzyc w kodzie bez kopiowania gotowego rozwiazania.

## Fundamenty

- [ ] Potrafie wyjasnic roznice miedzy frontendem, backendem, API i baza danych.
- [ ] Potrafie uruchomic projekt Node.js z terminala.
- [ ] Rozumiem `package.json`, skrypty npm i zaleznosci.
- [ ] Potrafie uzyc Gita do zapisania kolejnych etapow pracy.
- [ ] Potrafie debugowac prosty blad skladniowy i blad runtime.

## JavaScript i Node.js

- [ ] Rozumiem `let`, `const`, funkcje, obiekty, tablice i destrukturyzacje.
- [ ] Potrafie uzyc `map`, `filter`, `find`, `reduce` w prostych zadaniach.
- [ ] Rozumiem `Promise`, `async/await` i obsluge bledow przez `try/catch`.
- [ ] Potrafie odczytac i zapisac plik w Node.js.
- [ ] Rozumiem czym sa CommonJS i ES Modules.
- [ ] Potrafie stworzyc proste API na module `http`.

## HTTP i REST

- [ ] Rozumiem metody HTTP: GET, POST, PATCH, PUT, DELETE.
- [ ] Potrafie dobrac kod statusu do sytuacji.
- [ ] Rozumiem request body, query params, route params i headers.
- [ ] Potrafie zaprojektowac podstawowe endpointy REST.
- [ ] Potrafie zwrocic blad w przewidywalnym formacie.

## Express i TypeScript

- [ ] Potrafie stworzyc aplikacje Express.
- [ ] Rozumiem middleware i kolejnosc jego wykonywania.
- [ ] Potrafie wydzielic router, service i prosty repository.
- [ ] Potrafie skonfigurowac TypeScript w projekcie Node.
- [ ] Rozumiem typy, interfejsy, union types i generics na poziomie praktycznym.
- [ ] Potrafie zdefiniowac DTO i typ wyniku funkcji.

## NestJS

- [ ] Rozumiem modul, kontroler, provider i service.
- [ ] Potrafie wygenerowac zasob Nest CLI.
- [ ] Rozumiem dependency injection w NestJS.
- [ ] Potrafie stworzyc CRUD w module NestJS.
- [ ] Potrafie dodac DTO i globalna walidacje.
- [ ] Potrafie obsluzyc bledy przez wbudowane wyjatki NestJS.
- [ ] Rozumiem pipes, guards, interceptors i filters na poziomie zastosowan.

## Dane i architektura

- [ ] Potrafie zaprojektowac prosty model relacyjny.
- [ ] Potrafie uzyc Prisma do migracji i zapytan.
- [ ] Potrafie rozroznic logike aplikacyjna od logiki dostepu do danych.
- [ ] Potrafie uzyc transakcji tam, gdzie operacja musi byc atomowa.
- [ ] Potrafie dodac paginacje, filtrowanie i sortowanie.
- [ ] Potrafie przygotowac seed danych.

## Bezpieczenstwo i jakosc

- [ ] Potrafie hashowac hasla i nigdy ich nie zwracac z API.
- [ ] Potrafie zaimplementowac logowanie z JWT.
- [ ] Potrafie ograniczyc endpointy przez role.
- [ ] Potrafie skonfigurowac CORS, Helmet i rate limiting.
- [ ] Potrafie napisac test jednostkowy serwisu.
- [ ] Potrafie napisac test e2e dla endpointu.
- [ ] Potrafie skonfigurowac env, logowanie i dokumentacje OpenAPI.

## Projekt koncowy

- [ ] Mam dzialajace SkillForge API.
- [ ] Projekt uruchamia sie z README bez zgadywania.
- [ ] Baza danych startuje przez Docker albo jasna instrukcje lokalna.
- [ ] Wszystkie glowne endpointy maja walidacje i testy.
- [ ] Auth i role dzialaja zgodnie z zalozeniami.
- [ ] OpenAPI pokazuje glowne kontrakty.
- [ ] Projekt ma sensowna strukture modulow.
- [ ] Potrafie opowiedziec, dlaczego podjalem takie decyzje architektoniczne.
