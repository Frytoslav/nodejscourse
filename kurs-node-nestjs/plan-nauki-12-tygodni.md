# Plan nauki na 12 tygodni

To jest propozycja rytmu. Jezeli masz mniej czasu, rozciagnij plan. Lepiej zrobic kurs wolniej i uczciwie niz szybko przeklikac zadania.

## Tydzien 1

- Modul 00: sposob pracy.
- Modul 01: srodowisko, terminal, Git.
- Efekt: masz sandbox, notatki i umiesz uruchomic projekt.

## Tydzien 2

- Modul 02: JavaScript dla backendu.
- Modul 03: Node runtime, npm, asynchronicznosc.
- Efekt: prosta logika zadan, zapis i odczyt JSON.

## Tydzien 3

- Modul 04: HTTP bez frameworka.
- Efekt: pierwsze CRUD API na module `http`.

## Tydzien 4

- Modul 05: Express i warstwy.
- Efekt: API w Express z routerem, serwisem, repozytorium i centralna obsluga bledow.

## Tydzien 5

- Modul 06: TypeScript.
- Efekt: typowany backend sandbox i DTO.

## Tydzien 6

- Modul 07: NestJS fundamenty.
- Efekt: pierwszy projekt NestJS i CRUD w pamieci.

## Tydzien 7

- Modul 08: DTO, walidacja i bledy.
- Efekt: API odrzuca zle dane i ma jasne kontrakty.

## Tydzien 8

- Modul 09: Prisma i PostgreSQL.
- Efekt: kursy i lekcje zapisywane w bazie.

## Tydzien 9

- Modul 10: relacje, transakcje i logika domenowa.
- Efekt: zapisy na kurs, postep i pierwsze reguly biznesowe.

## Tydzien 10

- Modul 11: auth i bezpieczenstwo.
- Efekt: rejestracja, logowanie, JWT i role.

## Tydzien 11

- Modul 12: testy, konfiguracja i obserwowalnosc.
- Efekt: testy jednostkowe/e2e, env, health check i logi.

## Tydzien 12

- Moduly 13-14.
- Projekt koncowy: domkniecie SkillForge API.
- Efekt: dokumentacja API, Docker, README i finalny przeglad.

## Rytual tygodniowy

Na koniec kazdego tygodnia:

- zrob commit,
- dopisz notatke "czego sie nauczylem",
- wypisz 3 rzeczy, ktore nadal sa niejasne,
- uruchom wszystkie testy albo przynajmniej wszystkie endpointy z danego tygodnia,
- popraw README cwiczeniowego projektu, jesli instrukcja uruchomienia juz sie zmienila.
